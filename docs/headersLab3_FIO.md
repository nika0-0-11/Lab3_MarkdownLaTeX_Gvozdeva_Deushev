## Лабораторная работа №3. Работа с Markdown-разметкой и базовое использование LaTeX в документации проекта

####  Разметка Markdown — это облегченный язык форматирования для создания структурированных текстовых документов с помощью простых символов.

#### Markdown применяется: 

* Документация проектов (в README файлах на GitHub/GitLib/GitFlic)
* Веб-разработка
* Статические сайты 
* Системы управления версиями
* Заметки (в Obsidian, стандартных редакторах)

#### Разработчики используеют Markdown, чтобы:  

* Быстро оформлять документацию, README-файлы, заметки и сообщения
* Писать в обычном тексте, который остаётся читаемым даже без обработки
* Легко конвертировать текст в HTML, PDF и другие форматы
* Работать без громоздких редакторов, экономия время 
* Добиваться единообразного и чистого оформления текста
